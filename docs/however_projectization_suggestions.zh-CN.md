# promptfoo 私有化改造建议清单（60 条）

> 目标：在保留上游能力的前提下，将仓库改造成可长期维护的个人品牌项目。

## A. 品牌与命名（1-10）

1. 将 npm 包名调整为私有命名空间（例如 `@however-yir/howeverpromptfoo`）。
2. 保留 `promptfoo` CLI 入口，同时新增 `howeverpf` 别名命令。
3. 将 `package.json` 的 `author` 改为你的个人身份信息。
4. 将 `repository.url` 指向你的 GitHub 仓库地址。
5. 补充 `homepage` 与 `bugs.url` 字段，统一项目入口。
6. 将 README 头图替换为你的项目 Logo 与徽章组合。
7. 建立统一品牌文案（中文主文案 + 英文技术术语）。
8. 新增仓库描述模板，区分“个人发行版”和“上游社区版”。
9. 明确仓库 Topics（如 `llm-eval`、`red-team`、`prompt-testing`、`devsecops`）。
10. 建立视觉资源目录（Logo、Banner、封面图）并固定命名规范。

## B. 配置与敏感信息治理（11-20）

11. 新增 `.env.however.example` 作为统一本地配置模板。
12. 以 `OLLAMA_BASE_URL` 覆盖硬编码本地地址，默认改为可配置。
13. 新增 `HOWEVER_OPENAI_BASE_URL` 支持自建网关接入。
14. 新增 `HOWEVER_REDIS_URL` 作为缓存后端可选项。
15. 新增 `HOWEVER_DB_URL` 作为评测结果持久化地址占位。
16. 将 OTLP 追踪地址改为环境变量优先策略。
17. 增加密钥命名统一规则（`*_API_KEY`、`*_TOKEN`）。
18. 增加本地配置校验脚本，缺失关键变量时直接报错。
19. 增加 `.env` 误提交防护（pre-commit + gitignore 规则）。
20. 输出“开发/测试/生产”三套配置示例。

## C. 依赖与构建治理（21-30）

21. 审核并冻结核心依赖版本区间，减少漂移风险。
22. 将高风险依赖纳入 `overrides` 与定期审计机制。
23. 新增 `npm run deps:audit` 与 `npm run deps:update:minor` 脚本。
24. 将可选依赖按场景分组（云端、浏览器、语音、多模态）。
25. 补充 `engines` 兼容说明和最小 Node 版本测试矩阵。
26. 建立 `pnpm` 或 `npm ci` 一致化策略，避免锁文件分叉。
27. 为 CLI、Web、Docs 拆分独立构建耗时指标。
28. 增加 Docker 构建层缓存策略，缩短 CI 时间。
29. 增加依赖变更影响清单（功能、体积、许可证）。
30. 对外发布版本时固定 changelog 校验步骤。

## D. 代码结构与架构演进（31-40）

31. 在 `src/` 下增加 `core`, `adapters`, `apps` 分层边界说明。
32. 抽离 provider 适配层的公共重试与鉴权逻辑。
33. 将 redteam 策略注册改为插件化装配。
34. 将评测断言能力拆为“内置断言 + 扩展断言”双轨。
35. 为配置解析层增加 schema 版本迁移器。
36. 对核心命令执行链路补充状态机图。
37. 抽离 Web UI 与 CLI 的共享类型定义包。
38. 为外部集成（Langfuse、MCP、Browser）建立能力边界文档。
39. 将实验性能力统一放入 `experimental` 命名空间。
40. 新增“上游同步冲突处理”标准流程文档。

## E. 质量保障与安全（41-50）

41. 增加关键命令金丝雀测试（`init/eval/view/redteam`）。
42. 为 `OLLAMA_BASE_URL`、`OPENAI_BASE_URL` 增加格式校验测试。
43. 增加配置脱敏单元测试，防止日志泄漏密钥。
44. 补充端到端用例：本地模型、云模型、混合模型三场景。
45. 为数据库迁移脚本增加回滚验证步骤。
46. 建立 SAST + dependency scan + secret scan 三重检查。
47. 对 Web UI 增加无障碍基础检查（键盘导航/对比度）。
48. 对 CLI 输出增加稳定快照测试。
49. 建立性能基线（首个 eval 耗时、并发吞吐、内存峰值）。
