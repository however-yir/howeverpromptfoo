# 上游同步冲突处理流程

## 1. 同步前准备

1. 拉取上游最新主分支。
2. 记录本地发行版的关键改造点（配置、文档、脚本）。
3. 创建独立同步分支，例如：`sync/upstream-YYYYMMDD`。

## 2. 冲突处理策略

- 代码冲突：优先保留上游核心逻辑，再补回本地扩展点。
- 配置冲突：保留本地模板文件，不覆盖真实配置规范。
- 文档冲突：保留中文主文档，同时补充上游变化说明。

## 3. 回归检查

1. `npm run test:however`
2. `npm run deps:audit`
3. `npm run validate:however:env`

## 4. 合并准入

- 提供“上游变更摘要 + 本地适配点 + 风险项 + 回滚方案”。
- 通过 CI 后再合并至主分支。
